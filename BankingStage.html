Banking Stage
Banking Stage는 블록 생성 과정을 담당하는, TPU에서 가장 중요한 단계입니다. 여기서 bank란 특정 블록 시점에서의 상태 스냅샷입니다.
병렬 처리
트랜잭션들이 병렬로 처리되며, 엔트리라 불리는 기록으로 묶입니다. 각 엔트리는 충돌이 없는 64건의 트랜잭션을 포함합니다.
Race Condition 방지
각 트랜잭션이 읽고 쓸 모든 계정 목록을 미리 명시해야 하므로, 검증자가 Race condition을 피하고 병렬 처리를 수행할 수 있습니다.
멀티 스레드
6개의 스레드가 동시에 트랜잭션을 처리합니다. 4개는 일반 트랜잭션, 2개는 투표 트랜잭션 전담입니다.
CPU 기반
모든 병렬 처리는 CPU 멀티코어를 통해 진행되며, 검증자에게 별도의 GPU 요구사항은 없습니다.
SVM (Solana Virtual Machine): 트랜잭션 실행은 SVM이 담당하며, 이는 eBPF 프로그램을 위한 JIT 컴파일 및 가상 머신 기능을 제공하는 rBPF 라이브러리를 솔라나가 포크해 만든 가상 머신입니다.
